# 三项目隔离重构设计

## 背景

`ucas_earth` 项目包含三个功能模块：**校园展示**（Campus）、**火星探索**（Mars）、**城市漫游**（City）。当前三个模块共享组件（Header、CesiumViewer 等）、共享数据层、共享工具函数，导致修改一个模块可能影响其他模块。

**目标**：通过目录隔离，实现三个模块完全独立，修改一个不影响其他。

**核心约束**：
- 只重构目录结构，不修改现有功能逻辑
- 不引入新的错误和 bug
- 保持 SPA 路由切换
- 保持 Vue 3 + TypeScript + Vite 技术栈

## 路由配置

```typescript
// src/app/router/index.ts
const routes = [
  { path: '/', name: 'home', component: () => import('../views/HomeView.vue') },
  { path: '/campus', name: 'campus', component: () => import('../../campus/CampusView.vue') },
  { path: '/mars', name: 'mars', component: () => import('../../mars/MarsView.vue') },
  { path: '/city', name: 'city', component: () => import('../../city/CityRoamingView.vue') },
]
```

## Cesium 初始化

每个子项目各自在自己的入口文件中设置 `Cesium.Ion.defaultAccessToken`：
- `campus/CampusView.vue` — 通过 `CesiumViewer.vue` 设置
- `mars/MarsView.vue` — 已有设置
- `city/CityRoamingView.vue` — 已有设置

`App.vue` 中的全局赋值将被移除。

## 迁移策略

**核心原则**：只移动文件，不修改代码逻辑。

**步骤**：
1. 创建目录结构
2. 移动文件（按依赖顺序）
3. 更新 import 路径（仅修改文件移动导致的路径变化）
4. 验证（运行 `npm run dev` 确保三个视图正常工作）

**不做的事**：
- 不重命名任何组件
- 不修改任何组件的 props/emits/逻辑
- 不引入新的抽象层
- 不修改 CSS 样式
